## Dead Letters + Model Cascade Bulletproofed

### Problem
Two Nexus alerts kept firing:
1. Dead Letter — `daily_briefing` for chief-of-staff: `404 No endpoints found for anthropic/claude-3.5-sonnet` (OpenRouter removed this endpoint)
2. Pipeline Health — "No new session_logs rows in last 4 hours" (false positive from low-traffic periods)

### Root Cause
- `MODEL_CASCADE` in `server/model-manager.ts` had `anthropic/claude-3.5-sonnet` as final fallback — deprecated by OpenRouter
- Kilo only fired on `402` credit-exhaustion errors, not general failures
- Groq was skipped for `top` tier agents (chief-of-staff) — so when all OpenRouter models failed, chief-of-staff had nothing to fall back to
- Session log alert used a 4-hour rolling count — any quiet period during active hours fired an alert

### Fixes (commit `294f34b`)

**`server/model-manager.ts`:**
- Cascade expanded to 4 models (last 2 free-tier to survive credit exhaustion):
  `gpt-4o → llama-4-scout:free → claude-sonnet-4 → google/gemini-2.0-flash-exp:free`
- Added `getDirectOpenAIClient()` — uses `OPENAI_API_KEY` → `api.openai.com` directly (bypasses OpenRouter)
- Kilo now triggers on **any** cascade failure, not just 402
- Groq restriction removed — all agent tiers can use it
- `openai_direct` added to providerHealth tracking

**`server/agents/scheduled-jobs.ts`:**
- Session log check: changed from "count in 4h rolling window" to "time since last log > 8h"
- Eliminates false positives from simply not chatting with an agent for a few hours

### Full Fallback Chain (8 shots)
