## Claude Code Session #4 — Gemini 2.5 Flash for Knowledge Hub (2026-03-30)

### What Shipped (commit `1a5548d`)

**File:** `server/file-extraction.ts`

**Clarification first:** Gemini Embedding 001 is a *text embedding* model — it does NOT read PDFs or images. What helps with file processing is **Gemini 2.5 Flash** (a multimodal generative model), which uses the same `GOOGLE_AI_API_KEY` already in Railway.

**Changes:**
- `extractTextFromImage()`: Gemini 2.5 Flash primary → GPT-4o fallback (94% cheaper for vision)
- `analyzeTextWithAI()`: Gemini 2.5 Flash primary → GPT-4o-mini fallback
- `reanalyzeFile()`: Gemini 2.5 Flash primary → GPT-4o/GPT-4o-mini fallback
- **New**: Scanned PDF path — if `pdf-parse` returns <50 chars (image-based PDF), sends raw PDF to Gemini vision. Reads tables, charts, forms that pdf-parse misses. Size-gated at 15MB to avoid Gemini inline limit.

**No new dependencies.** Uses direct REST calls to Gemini API (same pattern as embeddings.ts).

### Cost Impact
| Task | Before | After | Saving |
|------|--------|-------|--------|
| Image OCR | GPT-4o $2.50/1M | Gemini 2.5 Flash $0.15/1M | 94% |
| PDF analysis | GPT-4o-mini $0.15/1M | Gemini 2.5 Flash $0.15/1M | Same |
| Scanned PDFs | ❌ Fails silently | ✅ Gemini vision | New capability |
