## MemPalace Evaluation + BM25 + Benchmark Harness (2026-04-13, commit 86e8070)

### MemPalace Decision
Evaluated [MemPalace](https://github.com/MemPalace/mempalace) as potential memory library.
**Verdict: Do not adopt.** SB-OS memory stack is architecturally more advanced. MemPalace is Python-only, local-only, experimental with public retractions of benchmark claims.

Two ideas worth stealing identified:
1. Benchmark harness — SB-OS had zero retrieval quality measurement
2. Real BM25 — keyword arm was mislabelled (single-token ILIKE, not BM25)

### BM25 Upgrade (`server/memory/hybrid-retriever.ts`)
- Replaced `LOWER(content) LIKE '%term[0]%'` with PostgreSQL full-text search
- Now uses `plainto_tsquery('english', query) @@ to_tsvector('english', content)`
- Ranking via `ts_rank_cd(..., 32)` — cover-density, normalised [0–1]
- Handles stemming, stop words, multi-word phrases natively
- `keywordSearchMemories` exported for testing
- Scoring formula unchanged: `0.70 * bm25 + 0.15 * ebbinghaus_decay + 0.15 * importance`

### GIN Index (`server/memory/ensure-indexes.ts`)
- New file: `ensureMemoryIndexes()` creates `idx_agent_memory_content_fts` GIN index at startup
- `CREATE INDEX IF NOT EXISTS ... USING gin(to_tsvector('english', content))`
- Drizzle 0.45 doesn't support functional index expressions in schema; applied via raw SQL
- Wired into `server/index.ts` alongside FalkorDB/Pinecone startup checks

### Benchmark Harness (`server/__tests__/retrieval-eval.test.ts`)
- 30 SB-OS-domain fixture documents (all major system areas)
- 20 labelled (query → relevant_doc_ids) eval pairs
- Baseline scores: R@1=0.975, R@3=1.000, R@5=1.000, MRR=1.000
- Regression guard: fails CI if R@5 drops below 0.75
- 6 total tests including exact-term ranking, multi-word vs partial, importance/recency, zero-score for nonsense

### Deployed
- Commit `86e8070` pushed to `sayedbaharun/SBOS` main — Railway auto-deploys
- First startup will build GIN index on all existing `agent_memory` rows
- `server/storage.ts` — `recordJobRun()`, `getLastJobRun()`
- `server/agents/agent-scheduler.ts` — `runCatchUpJobs()`, persistence on all paths
- `package.json` — added `cron-parser`
