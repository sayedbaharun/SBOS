## Repo: syntheliq-engine (orchestrator)

### Root Cause of 502s
SyntheLIQ Engine (separate Railway service) was crashing on boot because OpenRouter had run out of credits. All `/api/syntheliq/*` proxy calls from the website were failing → lead status PATCH, content approvals, settings saves all broken.

### Fixes Applied
1. **Kilo (EPI key) added to Railway env** — `KILOCODE_API_KEY` added, fallback chain now: Kilo → OpenAI → Google
2. **Agent schedules cleared** — all cron jobs disabled so engine boots clean without spending tokens. Only runs when manually triggered.
3. **Proxy body fix** — added `on.proxyReq` callback to serialise `req.body` onto outgoing proxy requests for all mutating methods (PUT/PATCH/POST). Without this, body was stripped → all PATCH calls to engine returned 502.
4. **Engine running** — verified `ok` status with 22 agents ready and idle post-fix.
