## Wave 5.2 — Delegation Execution Gap Fixed

**Commit:** `4760128` — pushed to `main`, Railway auto-deploying.

### Problem
Every delegation surface except the Chief-of-Staff in-chat tool was creating orphan `agent_tasks` rows with `status='pending'` that nothing ever executed. Sayed observed Nexus saying "task sent, working on it" with no actual agent activity.

### Root cause
`delegateFromUser()` at `server/agents/delegation-engine.ts:312-358` inserted the row and returned without calling `executeAgentTask()`. Affected surfaces:
- `/delegate` Telegram command
- `POST /api/tasks/:id/delegate-to-agent`
- `POST /api/agents/:slug/delegate`
- NL bar `delegate_task`
- MCP `delegate_task`
- Event bus fan-out to subscribed agents
- Proactive morning loop auto-delegations

### Fix
Added 12 lines in `delegation-engine.ts` after `messageBus.sendDelegation()` to fire-and-forget `executeAgentTask(task.id)` via lazy dynamic import (avoids circular dep with `agent-task.ts`). Errors caught + logged with `taskId` and `agentSlug`.

### Knock-on effects
- Wave 2 auto-approve policy engine (`server/agents/approval-policy-evaluator.ts`) starts working — was inert because no deliverables were being produced through these paths
- Proactive morning loop now actually runs the tasks it auto-delegates
- Event bus subscriptions become a real signal pipeline, not a write-only ledger
