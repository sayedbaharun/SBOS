**Root cause confirmed**: NOT a simple memory overflow but concurrent startup ops blocking the event loop + competing for pg pool. `getOrBuildIndex()` (BM25) was loading ALL active docs at boot while seeds and Qdrant sync ran in parallel, freezing the event loop. The 15s `Promise.race` timeout never fired because SIGKILL/event-loop-block killed the process before any timer callback could run. (Railway Hobby sliders already maxed at 8 vCPU / 8 GB ‚Äî no headroom to add.)

**Fix shipped (commit `948234e`)**:
- Removed eager BM25 build at boot ‚Äî now builds lazily on first `/api/docs/search` call
- Paginated `rebuildFromStorage` in `server/bm25.ts` ‚Äî 100 docs/page, `Array.isArray` guards fix `(doc.tags || []).join is not a function` warning
- Serialized `seedDefaultEventSubscriptions` + `seedTravelKnowledge` + `seedVenturesKnowledge` with `await` before `initializeScheduler` (were fire-and-forget racing the scheduler)
- Paginated `bulkSyncDocsToQdrant` in `server/memory/kb-qdrant.ts` ‚Äî 50 docs/page, no full result set in RAM
- Added `SKIP_QDRANT_BULK_SYNC` env-var kill switch
